# Test Roadmap — OpenCart Automated UI Testing

## Goal

Build and maintain a stable, readable, and scalable automated UI test suite for OpenCart.

---

## Phase 1 — Core automation (Current)
- Automate main customer journeys
- Establish Page Object Model structure
- Ensure tests are readable and repeatable

---

## Phase 2 — Coverage expansion
- Add more negative and edge cases
- Improve validation of error handling
- Add data-driven test scenarios

---

## Phase 3 — Stability improvements
- Reduce flaky tests
- Improve waits and synchronisation
- Refactor duplicated logic

---

## Phase 4 — Execution improvements
- Optional multi-browser execution
- Better reporting and screenshots
- CI-ready execution setup

---

## Maintenance

- Update tests when UI behaviour changes
- Keep documentation in sync with test coverage
